【构建想法】

在我没有建博客之前一直都有种想法就是将我在 X 上发的各种废话/心情/灵感/感悟/吐槽这些非常个人化的片段也存在个人网站上。 当时决定使用 blowfish 这个主题是因为看到了它在 shortcode 中设计的 timeline 这个功能， 在我看到一位博主 Ivon的部落格 的关于「动态墙」的设计后觉得这样陈列个人想法也挺合适的， 所以我的废话一部分会以 卡片形式转存到本网站的某 space 的 timeline 中，一部分会同步发在这个动态列表里。

【阅前声明】

本列表是及其私人化的表达，并且因为内容无法概括所以不能给出观前警告 ，同时观后请勿随意揣测立场，若有想法支持友好交流。

仅供娱乐，谨慎阅读

    最近一直在弄这个配置，然后我从一开始就有个疑问，那就是为什么我在参数文件中设置了 colorScheme 这个参数为「牛油果色」，而我只能在本地 localhost 的网页上看到，一旦推送到 .io 网页上就变成了默认的的蓝色。搞这个也有小半月了，虽然这个主题颜色差异也正好给我一个参考，能够让我区分哪个是我正在调整的本地页面，哪个是远端网站，但是我还是心存疑问。刚刚配置评论区块时看到有成功应用了 avocado 颜色的主题博客，我就更奇怪了，难道是我拼错了单词？是 avocardo 吗？还是那个自动化 action 有问题？（这个文件是我借鉴的，实际就是我拿来就用了还没看懂）随后我又去参考文档上非常仔细地瞅了好几遍。最后发现，我把 colorScheme = “avocado” 设置为了 Avocado。就这么一个字母的大写，让我疑问了很多天，甚至我都在想等我配置差不多的时候我就去提 issue 看看。最后我还是想知道为什么 hugo 的本地 server 能正常地渲染出来这个不是默认主题的颜色设置。刚刚我在成功纠正了这个大小写之后，看着绿色的网站又突然觉得那个蓝色的主页也挺顺眼的。哎，人。
